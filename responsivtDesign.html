Responsivt design

Publisert 05.10.2018.

Med responsivt design, snakker vi i denne sammenheng om at nettstedene vi lager skal respondere på skjermstørrelsen til klient-enheten. Det er flere måter å oppnå dette på, som media query, media type, responsive størrelser (for eksempel %, vw, vh, osv). Det absolutt letteste er å starte med å stile mobilversionen. I denne artikkelen viser jeg frem noen eksempler fra min egen kode, samt noen gode eksterne ressurser.


Media query

Media query er min favorittmåte å lage nettstedet responsivt, fordi det er enkelt og jeg kan ha all CSS i en enkel fil. Måten å gjøre det på, er å starte stilarket med å stile mobilversionen uten media query. Når det er gjort, kan man begynne å lage nettstedet responsivt ved å teste ved å gjøre nettleservinduet større. Gjør vinduet større helt til siden ikke ser bra ut, og lag så et brakepoint for den størrelsen. Her kommer media query inn i bildet. La oss si at nettstedet ser dårlig ut når man kommer opp i 600px. Fortsett i samme stilark, men sett all CSS inne i en media query sånn her; 

@media screen and (min-width: 600px) {
    .mainFlex {
        display: flex;
        flex-direction: row;
        justify-content: space-around;
        align-items: flex-start;
        flex-wrap: wrap;
    }

    .mainFlex article {
        max-width: 48%;
    }
}

Som illustrert i skjermdumpene under, resulterer denne koden i at forsiden viser to og to artikler når skjermvidden er større enn eller lik 600px.

Et annet eksempel er sidebaren. Denne har jeg stilert på litt flere forskjellige måter for å få den til å passe alle skjermstørrelser. Det lar seg lettere forklare med bilder, legg merke til hvordan sidebaren endrer både plassering av sidebaren og plassering av innholdet;


Media type

En annen måte å gjøre nettstedet responsivt er ved hjelp av flere stilark. Dette gjøres ved å lage et stilark for hver brakepoint. Stilarkene linkes inn i <head> ved å spesifisere viewport width på følgende måte;

<link rel="stylesheet" type="text/css"
media="screen and (max-width: 479px)"
href="stiler/stilarkMobilLiten.css">

<link rel="stylesheet" type="text/css"
media="screen and (min-width: 480px)"
href="stiler/stilarkMobil.css">

<link rel="stylesheet" type="text/css"
media="screen and (min-width: 600px)"
href="stiler/stilarkTab.css">

<link rel="stylesheet" type="text/css"
media="screen and (min-width: 900px)"
href="stiler/stilarkTabLand.css">

<link rel="stylesheet" type="text/css"
media="screen and (min-width: 1200px)"
href="stiler/stilarkDesktop.css">

Her er det viktig å linke inn stilarkene i riktig rekkefølge på grunn av cascading effekten (det som linkes til nederst blir gjeldende). Hvis stilarkMobil.css var plassert nederst, ville den overskrevet stilarkDesktop.css fordi en skjerm som er minimum 1200px også vil bli fanget opp av (min-width: 480px).


Responsive størrelser

Ved å bruke responsive størrelser vil man, i mange tilfeller, kunne unngå å lage en media query eller et ekstra stilark. Dette er fordi størrelser som %, vw (viewport width), vh (viewport height) vil endre seg automatisk etter størrelsen på skjermen/viewport.
